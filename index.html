GitHub Teljes Bemutató

Részletes magyarázatok, parancsok és működés

Mi az a GitHub?

A GitHub egy online platform, amely lehetővé teszi a fejlesztők számára, hogy kódot tároljanak, kezeljenek és másokkal együtt dolgozzanak rajta. A Git nevű verziókezelő rendszerre épül.

A verziókezelés azt jelenti, hogy minden változtatás mentésre kerül, így vissza lehet térni korábbi állapotokra.

Mi az a repository?

A repository (repo) egy projekt teljes mappája. Tartalmazza:

  • forráskód
  • képek, fájlok
  • verzióelőzmények

Minden projektnek saját repository-ja van.

Hogyan működik a Git?

  • Letöltöd a projektet: git clone
  • Módosítod a fájlokat
  • Hozzáadod: git add .
  • Mentés: git commit
  • Feltöltés: git push

Ez a folyamat folyamatosan ismétlődik.

Fontos Git parancsok

  • git init – új repo
  • git clone URL – letöltés
  • git status – állapot
  • git add . – hozzáadás
  • git commit -m "msg" – mentés
  • git push – feltöltés
  • git pull – frissítés
  • git branch – ágak
  • git checkout – váltás
  • git merge – egyesítés
  • git log – előzmények
  • git diff – változások
  • git reset – visszaállítás

Előnyök

  • Biztonságos mentés
  • Csapatmunka
  • Verziókövetés
  • Ingyenes használat

Hátrányok

  • Bonyolult tanulni
  • Sok parancs
  • Konfliktusok lehetnek (merge conflict)

GitHub funkciók

  • Pull Request – kód ellenőrzés
  • Issues – hibakezelés
  • Actions – automatizálás
  • Pages – weboldal készítés

Összegzés

A GitHub az egyik legfontosabb eszköz a modern fejlesztésben. Segít rendszerezni a munkát és lehetővé teszi a hatékony együttműködést.
